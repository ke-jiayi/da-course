# 数据学院网站重构改进（v2）- 产品需求文档

## Overview

- **Summary**: 基于参考网站 `https://data-c.pages.dev` 的设计理念和功能实现，对当前 `https://da-course-9yx.pages.dev` 进行全面的体验改进和内容优化。重点改进包括：Pyodide加载体验优化、首页内容重构、导航简化、课程内容深度增加、成就徽章系统完善。
- **Purpose**: 解决当前网站用户面临的主要问题：环境加载等待焦虑、内容深度不足、导航混乱、学习激励缺失。参考竞品网站的成功经验，提升整体专业性和学习价值。
- **Target Users**: 
  - 数据分析初学者（零基础）
  - Python编程入门学习者
  - 希望掌握数据技能的职场人士
  - 学生和自学者

## Goals

1. **提升加载体验**: 将Pyodide加载过程从"黑屏等待"改为有明确进度和提示的过程，减少用户流失
2. **重构首页展示**: 从营销型首页改为项目驱动的学习入口，直接展示10个实战项目
3. **简化导航结构**: 从10+个导航项精简为4-5个核心导航，降低认知负担
4. **增强课程内容**: 每个课程增加真实数据集示例和更多实战步骤，从"演示"变为"学习"
5. **完善激励系统**: 实现可见的成就徽章系统，为完成课程的用户提供奖励反馈
6. **参考网站对标**: 整体风格和交互模式向 `https://data-c.pages.dev` 对齐

## Non-Goals (Out of Scope)

- **不修改核心功能**: 不改变Python代码运行机制本身（Pyodide核心保持不变）
- **不重新设计UI风格**: 不改变当前的"cute"配色和圆角风格，只优化布局和内容
- **不添加后端功能**: 不引入真实的用户系统或数据库，保持纯前端静态部署
- **不改变路由结构**: 保持现有的10个课程路由不变，只修改组件内部实现
- **不添加新的课程**: 保持当前10门课程的数量不变

## Background & Context

### 参考网站分析（目标）

`https://data-c.pages.dev` 的核心特点：

1. **首页简洁高效**: 
   - 顶部大标题："Pandas 数据分析实战训练营"
   - 副标题强调："10个精选实战项目，从入门到进阶，完全在浏览器中运行代码"
   - 三大卖点：真实数据集 · 实时运行代码 · 循序渐进 · 徽章认证
   - 直接展示10个项目卡片网格

2. **项目卡片设计**:
   - 大图标（emoji）居中展示
   - 难度标签（入门/进阶/高级）
   - 时间标签（30分钟/45分钟/60分钟）
   - 使用的数据集名称
   - "开始学习"按钮

3. **内容深度**:
   - 每个项目围绕真实数据集展开
   - 代码示例直接使用该数据集进行分析
   - 图表和分析结果即时可见

### 当前网站问题分析（现状）

1. **Pyodide加载问题**:
   - 首次进入课程页面，用户看到一个简单的"正在加载"提示
   - 没有进度指示、没有预估时间、没有说明正在加载什么
   - 约30秒的等待时间内用户可能会离开页面
   - 相关文件: [pyodideService.ts](file:///workspace/src/services/pyodideService.ts), 各课程组件

2. **首页信息过载**:
   - 页面过长：平台介绍 → 课程分类（10个）→ 最新动态 → 关于我们
   - 与参考网站的"单屏即达"形成对比
   - 用户需要多次滚动才能找到实际可操作的课程入口
   - 相关文件: [Home.tsx](file:///workspace/src/components/Home.tsx)

3. **导航混乱**:
   - Header中有10个导航项：首页、课程、代码编辑器、练习、测试、成就、排行榜、通知、设置、思维测试
   - 与参考网站的"极简导航"（仅首页+可能的徽章系统）形成对比
   - 用户难以分辨主次
   - 相关文件: [Header.tsx](file:///workspace/src/components/Header.tsx)

4. **课程内容偏浅**:
   - 许多课程仅有基础概念介绍和简单代码
   - 缺乏真实数据集的深入分析案例
   - 步骤分解不够细致
   - 相关文件: [DataVisualization.tsx](file:///workspace/src/components/DataVisualization.tsx), [MachineLearning.tsx](file:///workspace/src/components/MachineLearning.tsx) 等10个课程文件

5. **成就系统不可见**:
   - 有 [Achievements.tsx](file:///workspace/src/components/Achievements.tsx) 页面但与课程没有关联
   - 用户完成课程后没有获得徽章的反馈
   - 与参考网站"徽章认证"卖点形成对比

### 技术架构

- **框架**: React + TypeScript + Vite
- **核心服务**: Pyodide（浏览器端Python运行环境）
- **样式**: Tailwind CSS（自定义主题色）
- **部署**: Cloudflare Pages
- **状态管理**: React hooks + localStorage

所有修改都在现有技术框架内完成，不需要引入新依赖。

## Functional Requirements

### FR-1: Pyodide加载体验优化

**描述**: 将Pyodide加载过程从简单提示改为有进度条和阶段说明的完整加载界面。

**具体要求**:
1. 在所有使用Pyodide的页面中（10个课程 + PythonPlayground），统一加载体验
2. 显示分阶段加载提示：
   - 阶段1："正在连接 Python 运行环境"（脚本下载）
   - 阶段2："正在加载核心库 (pandas/numpy)"（包安装）
   - 阶段3："正在配置可视化工具"（matplotlib设置）
   - 阶段4："准备就绪！"（完成）
3. 显示可视化进度条（百分比）
4. 显示已用时间和预估剩余时间
5. 加载完成后自动平滑过渡到主内容（淡出动画）
6. 如果加载失败，显示友好的错误提示和"重试"按钮
7. 提供"加载中可以做什么"的小贴士（如：阅读下面的课程介绍）

**涉及文件**:
- [pyodideService.ts](file:///workspace/src/services/pyodideService.ts) - 服务层需要暴露加载进度事件
- 所有课程组件（10个）- 需要统一使用新的加载UI组件
- [PythonPlayground.tsx](file:///workspace/src/components/PythonPlayground.tsx)

### FR-2: 创建统一的加载组件

**描述**: 创建一个可复用的 `PyodideLoader` 组件，供所有需要Python环境的页面使用。

**具体要求**:
1. 组件接收 props：当前阶段(0-4)、进度百分比、是否有错误、重试回调
2. 响应式设计，适配移动端和桌面端
3. 与当前网站配色风格一致
4. 包含有趣的加载动画（如跳动的代码行、旋转的蛇图标等）

### FR-3: 首页重构

**描述**: 将 [Home.tsx](file:///workspace/src/components/Home.tsx) 从营销型页面改为参考网站风格的项目驱动入口。

**具体要求**:
1. 顶部：大标题 + 副标题（强调"10个实战项目"、"浏览器端运行"）
2. 三大卖点标签："真实数据集" · "实时运行代码" · "循序渐进" · "徽章认证"
3. 主体：10个项目卡片的网格布局
   - 大图标（使用当前已有的emoji）
   - 项目标题
   - 难度标签（入门/进阶/高级）+ 时间标签
   - 数据集名称
   - "开始学习"按钮
4. 删除或简化以下部分：
   - "最新动态"板块（3条新闻）
   - "关于我们"板块（成立时间、统计数据）
5. 保留：
   - "数据学院"品牌标识
   - 渐变背景的视觉风格

### FR-4: 导航简化

**描述**: 修改 [Header.tsx](file:///workspace/src/components/Header.tsx)，将10个导航项精简为4-5个核心入口。

**具体要求**:
1. 保留的导航项（桌面端）：
   - 首页
   - 课程（或改为"实战项目"）
   - 代码编辑器（Python Playground）
   - 徽章（成就系统入口）
   - 登录/用户菜单
2. 删除的导航项：
   - 练习（已整合在课程内）
   - 测试（已整合在课程内或作为课程的一部分）
   - 排行榜
   - 通知
   - 设置
   - 思维测试
3. 移动端菜单同样简化
4. 视觉风格保持不变（黄色背景按钮、圆角等）

### FR-5: 课程内容增强（所有10个课程）

**描述**: 为每门课程增加更深入的内容和真实数据集示例。

**通用改进要求（所有10个课程）**:
1. **知识要点板块增强**:
   - 从简单的概念列表改为"为什么重要"+"核心概念"+"常见误区"的结构
   - 每个知识点增加简短说明（1-2句话）
2. **动手练习增强**:
   - 现有简单代码保留
   - 增加2-3个递进式练习步骤（Step 1: 基础 → Step 2: 进阶 → Step 3: 挑战）
   - 每个步骤有独立的代码编辑器和参考答案
   - 步骤之间有进度指示（如："第 1 / 3 步"）
3. **真实数据集集成**:
   - 在代码中预置真实的示例数据（小数据集，如10-20行销售记录）
   - 数据集与课程主题相关
   - 代码示例直接操作这个数据集（而不是抽象的数字列表）

**各课程的具体数据集要求**:
1. **数据可视化** (retail_orders.csv): 电商订单数据，学习用图表展示销售趋势
2. **机器学习** (customer_features.csv): 客户特征数据，学习聚类和分类
3. **数据挖掘** (market_basket.csv): 购物篮数据，学习关联规则
4. **商业分析** (ab_test.csv): A/B测试结果数据，学习假设检验
5. **数据清洗实战** (retail_orders.csv): 有问题的订单数据，学习清洗技巧
6. **分组聚合分析** (retail_orders.csv): 订单数据，学习分组统计
7. **购物篮分析** (market_basket.csv): 购物记录，学习Apriori算法
8. **A/B测试分析** (ab_test.csv): 测试数据，学习显著性检验
9. **时间序列分析** (time_series_sales.csv): 月度销售数据，学习趋势预测
10. **异常值检测** (customer_features.csv): 客户数据，学习离群点检测

**涉及文件**: 所有10个课程组件（DataVisualization.tsx, MachineLearning.tsx, DataMining.tsx, BusinessAnalysis.tsx, DataCleaning.tsx, GroupAggregation.tsx, MarketBasket.tsx, ABTesting.tsx, TimeSeries.tsx, AnomalyDetection.tsx）

### FR-6: 徽章/成就系统完善

**描述**: 使 [Achievements.tsx](file:///workspace/src/components/Achievements.tsx) 成为一个有实际功能的页面，并与课程完成状态关联。

**具体要求**:

   - "渐入佳境"：完成5个课程（含2个进阶）
   - "数据分析大师"：完成全部10个课程

4. 徽章页面展示：
   - 已获得徽章（彩色，高亮显示）
   - 未获得徽章（灰色，显示获取条件）

6. 相关服务文件: [achievementService.ts](file:///workspace/src/services/achievementService.ts) - 如不存在则创建

### FR-7: 课程页面底部增加完成反馈

**描述**: 在每个课程页面底部添加完成标记功能。

**具体要求**:
1. 添加"我已完成本课程"按钮
2. 点击后：
   - 显示祝贺动画
   - 展示获得的徽章
   - 按钮变为"已完成 ✓"状态

4. 下次进入该课程时，显示"已完成"标记

## Non-Functional Requirements

### NFR-1: 性能

- Pyodide加载UI本身不能增加显著的页面加载开销（<50KB额外代码）
- 动画效果使用CSS实现，不使用重量级动画库
- 课程页面首次渲染时间保持 <2秒（不包含Pyodide自身加载时间）

### NFR-2: 兼容性

- 所有改进在Chrome, Firefox, Safari最新版正常工作
- 移动端响应式布局保持不变
- Cloudflare Pages部署流程保持不变

### NFR-3: 可维护性

- 新增组件遵循现有代码风格
- 使用TypeScript类型定义
- 所有课程组件保持相似的代码结构（便于后续统一维护）

### NFR-4: 用户体验

- 所有按钮点击有明确反馈（状态变化/动画/颜色变化）
- 加载过程中用户可以阅读页面其他内容（非阻塞式UI）
- 错误信息用中文编写，友好易懂

## Constraints

- **技术**: React + TypeScript + Vite + Tailwind CSS，不引入新框架
- **部署**: Cloudflare Pages，纯静态站点
- **数据存储**: 仅使用 localStorage，不引入后端数据库
- **时间**: 所有修改在现有代码基础上完成，不做架构级重构

## Assumptions

1. 用户的浏览器支持现代ES6+特性（Pyodide本身要求这一点）
2. 用户有基本的互联网连接速度（Pyodide下载约10-20MB数据）
3. localStorage可用且有至少1KB空间（用于存储徽章状态）
4. 参考网站 `https://data-c.pages.dev` 的设计风格可以借鉴但不应复制（保持原创）

## Acceptance Criteria

### AC-1: Pyodide加载体验
- **Given**: 用户首次访问任意课程页面或Python编辑器
- **When**: Pyodide正在初始化
- **Then**: 
  1. 用户看到分阶段的加载提示（至少3个阶段）
  2. 看到进度条和百分比
  3. 看到已用时间
  4. 加载完成后平滑切换到主内容
- **Verification**: `human-judgment` + `programmatic`（检查服务层是否有进度事件）
- **Notes**: 参考参考网站的加载提示风格

### AC-2: 首页重构完成
- **Given**: 用户访问首页
- **When**: 页面加载完成
- **Then**: 
  1. 看到大标题"数据学院"和副标题（描述10个实战项目）
  2. 看到4个卖点标签：真实数据集、实时运行代码、循序渐进、徽章认证
  3. 看到10个项目卡片，卡片包含：图标、标题、难度、时长、数据集名称、"开始学习"按钮
  4. **不再**看到"最新动态"和"关于我们"板块
- **Verification**: `human-judgment`

### AC-3: 导航简化
- **Given**: 用户在任意页面
- **When**: 查看顶部导航栏
- **Then**: 
  1. 看到不超过5个导航项（首页、课程、代码编辑器、徽章、登录）
  2. **不再**看到：练习、测试、排行榜、通知、设置、思维测试
  3. 移动端菜单同样只有这些核心项
- **Verification**: `human-judgment`

### AC-4: 课程内容增强（以数据可视化为样例）
- **Given**: 用户进入"数据可视化"课程
- **When**: 浏览"知识要点"和"动手练习"部分
- **Then**: 
  1. "知识要点"有结构化的说明（为什么重要/核心概念/常见误区）
  2. "动手练习"至少有3个递进步骤，每个步骤有独立编辑器
  3. 代码中使用真实的零售订单数据集（retail_orders.csv风格）
  4. 每个步骤可以独立运行并看到结果
- **Verification**: `human-judgment`

### AC-5: 所有课程内容增强
- **Given**: 用户访问任意一门课程
- **Then**: 该课程满足AC-4的所有要求（结构化知识要点 + 3步递进练习 + 真实数据集）
- **Verification**: `human-judgment`（抽查3门不同难度的课程）

### AC-6: 徽章系统
- **Given**: 用户访问"徽章"页面
- **When**: 页面加载完成
- **Then**: 
  1. 看到13个徽章（10个课程徽章 + 3个进阶徽章）
  2. 已获得的徽章彩色高亮显示，未获得的灰色显示
  3. 显示进度统计（如："已获得 2 / 13 徽章"）
  4. 每个徽章有图标、名称、描述
- **Verification**: `human-judgment` + `programmatic`（检查localStorage是否正确存储）

### AC-7: 课程完成标记
- **Given**: 用户在课程页面底部
- **When**: 点击"我已完成本课程"
- **Then**: 
  1. 显示祝贺动画或提示
  2. 按钮状态变为"已完成 ✓"
  3. 对应徽章在"徽章"页面显示为已获得
  4. 刷新页面后状态保持不变

### AC-8: 代码编辑器工作正常
- **Given**: 所有改进完成后
- **When**: 用户在任意课程页面或Python编辑器中编写并运行代码
- **Then**: 
  1. 代码正常执行，输出显示正确
  2. matplotlib图表正常渲染（如果代码中有绘图）
  3. 错误代码显示友好的错误提示（而非崩溃）
- **Verification**: `human-judgment`（手动测试3段代码：简单print、pandas操作、matplotlib绘图）
