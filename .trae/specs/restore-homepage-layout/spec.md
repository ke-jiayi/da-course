# 恢复网站首页布局 - 产品需求文档

## 概述
- **摘要**：恢复TRCE网站的首页四板块卡片布局，同时保留每个板块内部的交互式Python在线编程功能，确保首页布局与最初的马卡龙3D可爱风格完全一致，而每个板块详情页保留现有的交互式编程功能。
- **目的**：恢复网站的原始结构和风格，同时保留已实现的交互式编程功能，提供更好的用户体验。
- **目标用户**：零基础数据分析入门学习者，对Python编程和数据分析感兴趣的初学者。

## 目标
- 恢复首页四板块卡片布局（数据可视化、机器学习、数据挖掘、商业分析）
- 保留每个板块内部的交互式Python在线编程功能
- 确保首页卡片作为入口，点击跳转到对应板块的详情页
- 保持马卡龙3D可爱简约风格的视觉统一
- 保留现有的内容和功能，只调整页面结构

## 非目标（范围外）
- 不修改现有板块的内容和功能
- 不添加新的功能模块
- 不改变现有的设计风格

## 背景与上下文
- 网站之前已经实现了交互式Python在线编程功能
- 网站最初的首页布局是四板块卡片样式，具有马卡龙3D可爱风格
- 现在需要恢复这种布局，同时保留已实现的功能

## 功能需求
- **FR-1**：恢复首页四板块卡片布局，包括数据可视化、机器学习、数据挖掘、商业分析四个独立卡片
- **FR-2**：每个卡片保留3D可爱封面图、板块标题、简介文字
- **FR-3**：点击卡片进入对应板块的详情页
- **FR-4**：每个板块详情页保留交互式Python在线编辑器功能，支持用户自由输入/修改代码、点击运行、查看结果、自动纠错教学
- **FR-5**：首页只保留入口卡片，不直接放编辑器

## 非功能需求
- **NFR-1**：视觉风格保持一致，采用马卡龙3D可爱简约风格
- **NFR-2**：操作流畅，页面加载速度快
- **NFR-3**：响应式设计，适配不同屏幕尺寸

## 约束
- **技术**：基于现有的React + TypeScript + Tailwind CSS技术栈
- **内容**：保留现有的内容和功能，只调整页面结构
- **风格**：保持现有的马卡龙3D可爱简约风格

## 假设
- 网站已有的交互式Python在线编程功能可以正常工作
- 网站的路由系统可以支持首页到详情页的跳转

## 验收标准

### AC-1：首页四板块卡片布局
- **给定**：用户访问网站首页
- **当**：用户查看首页内容
- **则**：用户能够看到数据可视化、机器学习、数据挖掘、商业分析四个独立卡片，每个卡片都有3D可爱封面图、板块标题和简介文字
- **验证**：`human-judgment`

### AC-2：卡片点击跳转
- **给定**：用户访问网站首页
- **当**：用户点击任意板块卡片
- **则**：用户被跳转到对应板块的详情页
- **验证**：`programmatic`

### AC-3：交互式Python在线编辑器功能
- **给定**：用户访问板块详情页
- **当**：用户在编辑器中输入/修改代码并点击运行
- **则**：用户能够看到代码运行结果，系统提供自动纠错教学
- **验证**：`programmatic`

### AC-4：视觉风格一致性
- **给定**：用户浏览网站
- **当**：用户查看首页和详情页
- **则**：所有页面都保持马卡龙3D可爱简约风格，视觉统一
- **验证**：`human-judgment`
