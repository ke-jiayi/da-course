# 修复课程详情页 Python 代码报错 - 产品需求文档

## 概述
- **摘要**：修复数据可视化、数据挖掘、商业分析三个课程详情页的报错问题，问题是 Vite+Babel 在编译 TSX 组件时解析了 Python 代码，导致报 Unexpected token。同时按照用户要求的格式重新组织教学内容。
- **目的**：解决网站编译错误，使课程详情页能够正常访问，同时优化教学内容的组织结构。
- **目标用户**：零基础数据分析入门学习者，对 Python 编程和数据分析感兴趣的初学者。

## 目标
- 修复三个课程详情页的编译错误
- 确保 Python 代码在 TSX 组件中正确显示，不被 Babel 错误解析
- 按照用户要求的格式重新组织教学内容
- 保持交互式 Python 编辑器功能
- 确保教学内容零基础友好，易于理解

## 非目标（范围外）
- 不修改首页布局和导航结构
- 不改变现有的设计风格
- 不修改机器学习课程详情页（已经正常工作）

## 背景与上下文
- 网站已经恢复了首页四板块卡片布局
- 机器学习课程详情页工作正常，其他三个课程详情页出现编译错误
- 错误原因是 Python 代码直接写在 TSX 组件中，Babel 尝试解析其中的特殊字符

## 功能需求
- **FR-1**：修复数据可视化课程详情页的编译错误
- **FR-2**：修复数据挖掘课程详情页的编译错误
- **FR-3**：修复商业分析课程详情页的编译错误
- **FR-4**：按照用户要求的格式重新组织教学内容：
  - 学习目标（3 条内）
  - 前置知识说明
  - 3-4 个小节：知识点讲解 + 可运行 Python 示例 + 结果说明
  - 交互式 Python 编辑器（空白可输入，默认预填可运行代码）
  - 1 个课后练习任务

## 非功能需求
- **NFR-1**：教学内容用通俗的中文讲解，零基础友好
- **NFR-2**：代码示例清晰易读
- **NFR-3**：页面风格保持现有浅色系、可爱简约风格
- **NFR-4**：网站能够正常编译和运行，无错误

## 约束
- **技术**：基于现有的 React + TypeScript + Tailwind CSS 技术栈
- **内容**：按照用户指定的格式进行重新组织
- **风格**：保持现有的设计风格

## 假设
- 机器学习课程详情页的实现方式是正确的
- Python 代码可以通过正确的方式在 TSX 组件中显示
- 用户要求的教学内容格式是合理的
